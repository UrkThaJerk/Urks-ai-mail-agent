# Urks-ai-mail-agent

Supports the original mail workflow, a prototype video editing agent, a social media publishing agent, a collaborative four-agent mode, and a **web UI** so you can run any agent from your browser.

## Web UI

Start the Flask server and open the dashboard in your browser:

```bash
python web_app.py
```

Then navigate to <http://localhost:5000>.

You can also start it through the main entrypoint:

```bash
URKS_AGENT_TYPE=web python agent.py
```

Optional environment variables:

| Variable    | Default | Description |
|-------------|---------|-------------|
| `WEB_PORT`  | `5000`  | Port to listen on |
| `WEB_DEBUG` | `false` | Enable Flask debug/reload mode |

> **Note:** Gmail OAuth and YouTube OAuth flows require `credentials.json` and
> `token.json` to already exist on disk before using the mail or social agents
> through the web UI (same as the CLI).

## Mail agent

```bash
python agent.py
```

## Video editing agent

Use `URKS_AGENT_TYPE=video` to switch the entrypoint into video editing mode.

```bash
URKS_AGENT_TYPE=video \
VIDEO_INPUT_PATH=/absolute/path/to/input.mp4 \
VIDEO_EDIT_INSTRUCTIONS='detect scenes, apply sepia, add text "Intro", watermark "Urks", sync audio and export to /absolute/path/to/output.mp4' \
python agent.py
```

For batch processing, set `VIDEO_BATCH_SPEC` to a JSON file containing a list of `video_path`, `instructions`, and optional `output_path` objects.

## Streaming platform authentication

`streaming_auth.py` handles login to Twitch, YouTube, and Kick so the agents
can download authenticated VODs and post clips on a trending schedule.

### Twitch (OAuth Device Code — no browser required in CI)

1. Create a Twitch application at <https://dev.twitch.tv/console/apps> and note
   the **Client ID** and **Client Secret**.
2. Set the environment variables, then run:

```bash
TWITCH_CLIENT_ID=your_client_id \
TWITCH_CLIENT_SECRET=your_client_secret \
python streaming_auth.py twitch
```

Follow the on-screen prompt (visit the URL and enter the code).  
The token is saved to `~/.urks_streaming_tokens.json` and refreshed
automatically on subsequent runs.

To use in Python:

```python
from streaming_auth import get_credentials
creds = get_credentials("twitch")          # refreshes silently if needed

from video_agent import VideoEditingAgent
agent = VideoEditingAgent()
project = agent.load_from_url(
    "https://www.twitch.tv/videos/2800009020",
    instructions="detect scenes and make clips",
    download_dir="/tmp/twitch",
    credentials=creds,
)
```

### YouTube (OAuth 2.0 installed-app flow)

1. Download your OAuth 2.0 client secrets JSON from
   <https://console.cloud.google.com/> and save it (e.g. `youtube_client_secrets.json`).
2. Authenticate:

```bash
YOUTUBE_CLIENT_SECRETS_FILE=youtube_client_secrets.json \
python streaming_auth.py youtube
```

A browser window opens for consent.  The token is written to
`youtube_token.json` and re-used on future runs.

### Kick (cookie-based)

Kick has no public OAuth API.  Export a Netscape-format cookies file from a
logged-in browser session using an extension such as
[Get cookies.txt LOCALLY](https://chrome.google.com/webstore/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc),
then register it:

```bash
KICK_COOKIES_FILE=/path/to/kick_cookies.txt \
python streaming_auth.py kick
```

Or pass it programmatically:

```python
from streaming_auth import get_credentials
creds = get_credentials("kick", kick_cookies_file="/path/to/kick_cookies.txt")
```

### Force re-authentication

```bash
python streaming_auth.py twitch --force
```

### Token store location

Tokens are stored at `~/.urks_streaming_tokens.json` by default.  Override
with the `STREAMING_TOKEN_STORE` environment variable.

## Social media publishing agent

Use `URKS_AGENT_TYPE=social` to upload clips to YouTube, TikTok, and Twitter/X
on a trending schedule generated by the LLM.

```bash
URKS_AGENT_TYPE=social \
SOCIAL_CLIP_PATHS='["/tmp/twitch/clips/vod_scene_1.mp4","/tmp/twitch/clips/vod_scene_2.mp4"]' \
SOCIAL_TOPIC="gaming highlights" \
SOCIAL_PLATFORMS="youtube,twitter" \
SOCIAL_UPLOAD_NOW=true \
OPENAI_API_KEY=sk-... \
python agent.py
```

Set `SOCIAL_OUTPUT_PATH` to save the publish result JSON to a file.  
Set `SOCIAL_UPLOAD_NOW=false` (default) to only generate the schedule without uploading.

Required credentials per platform:

| Platform  | Environment variables |
|-----------|-----------------------|
| YouTube   | `YOUTUBE_CLIENT_SECRETS_FILE`, `YOUTUBE_TOKEN_FILE` |
| TikTok    | `TIKTOK_ACCESS_TOKEN` |
| Twitter/X | `TWITTER_API_KEY`, `TWITTER_API_SECRET`, `TWITTER_ACCESS_TOKEN`, `TWITTER_ACCESS_SECRET` |

## Collaborative four-agent mode

Use `URKS_AGENT_TYPE=collective` to run four separate specialists:

- a mail specialist
- a video specialist
- a social media specialist
- a strategy specialist

Each agent works on the objective independently first, then reviews the other agents' work so the group can learn collectively.

```bash
URKS_AGENT_TYPE=collective \
COLLECTIVE_OBJECTIVE='Improve the mail and video workflows together' \
COLLECTIVE_CONTEXT='Keep the agents separate, but let them share feedback after their first pass.' \
python agent.py
```

Set `COLLECTIVE_OUTPUT_PATH` to save the JSON result to a file.
